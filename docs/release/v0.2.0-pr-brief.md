# Steno v0.2.0 PR Brief

## Summary

`main` currently corresponds to public release `v0.1.10`. This branch, `feat/functional-followups`, packages the 0.2 release candidate: the full macOS redesign, the recent engine and cleanup fixes, the compatibility-matrix/model-guidance work, the release-eval harness, and the supporting doc refresh for a major public release.

## User-Facing Highlights

- Full shell redesign across Record, History, Settings, onboarding, and the floating overlay
- Richer local transcription ingestion from `whisper.cpp`
- Better local cleanup behavior for repair phrases, prompt contamination, and no-speech handling
- Compatibility-matrix-backed model guidance instead of vague “higher-end Apple Silicon” copy
- Formal release-eval path with a currently validated exact row: `m5-pro / 64GB / large-v3-turbo`

## Known Limitations

- Release-eval validation is row-specific, not universal hardware proof.
- `commandPassthroughAccuracy` remains `not_evaluable` on the current canonical corpus because the raw benchmark pass never preserved a leading slash.
- Manual macOS sanity is still tracked separately from the metric gate and should be completed if a full human release checklist is required before merge.
- Repair cleanup is materially stronger, but raw repair-marker preservation is still not something to oversell as “perfect.”
